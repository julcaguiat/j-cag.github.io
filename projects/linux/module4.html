Module 4: Linux Networking & Connectivity Troubleshooting

Overview

Network connectivity is a fundamental requirement for Linux servers operating in enterprise environments. Whether supporting Active Directory integrations, web services, cloud workloads, monitoring systems, virtualization platforms, or security operations, Linux administrators must be capable of diagnosing and resolving network-related issues quickly.

This module focuses on Linux networking fundamentals, interface configuration, routing, DNS troubleshooting, connectivity validation, firewall analysis, port investigation, and common troubleshooting methodologies used in production environments.

Learning Objectives

  • Understand Linux networking architecture.
  • Identify and configure network interfaces.
  • Verify IP addressing and routing.
  • Troubleshoot DNS resolution issues.
  • Test network connectivity.
  • Analyze open ports and listening services.
  • Capture and inspect network traffic.
  • Troubleshoot firewall-related connectivity problems.
  • Diagnose common enterprise network issues.
  • Perform systematic network troubleshooting.

Viewing Network Interfaces

Display Interface Information

ip addr

Example output:

2: ens18:
    inet 10.10.100.15/24

Display Interface Summary

ip -br addr

Example:

lo      UNKNOWN 127.0.0.1/8
ens18   UP      10.10.100.15/24

Legacy Method

ifconfig

Modern Linux distributions use the ip command instead of ifconfig.

Understanding Interface States

State Meaning
UP Interface operational
DOWN Interface disabled
UNKNOWN Virtual or loopback interface
NO-CARRIER No physical link detected

Enable and Disable Interfaces

Bring Interface Up

sudo ip link set ens18 up

Bring Interface Down

sudo ip link set ens18 down

Verify Status

ip link show ens18

Viewing IP Address Configuration

Display IPv4 information:

ip -4 addr

Display IPv6 information:

ip -6 addr

Assigning IP Addresses

Temporary Static Address

sudo ip addr add 10.10.100.50/24 dev ens18

Remove Address

sudo ip addr del 10.10.100.50/24 dev ens18

Verify Configuration

ip addr show ens18

Routing Fundamentals

A routing table determines where network traffic is forwarded.

View Routing Table

ip route

Example:

default via 10.10.100.1 dev ens18
10.10.100.0/24 dev ens18

Understanding Routes

Route Purpose
Default Route Gateway to external networks
Connected Route Directly attached network
Static Route Manually configured route
Dynamic Route Learned via routing protocol

Add Static Route

sudo ip route add 10.10.30.0/24 via 10.10.100.1

Testing Connectivity

Ping

Verify network reachability:

ping 10.10.100.1

Example:

64 bytes from 10.10.100.1

Limit Ping Count

ping -c 4 8.8.8.8

Measure Latency

ping google.com

Example:

time=18 ms

Path Analysis

traceroute

Displays the network path to a destination.

Install:

sudo apt install traceroute

Run:

traceroute 8.8.8.8

tracepath

Alternative utility:

tracepath 8.8.8.8

DNS Fundamentals

DNS translates hostnames into IP addresses. 

wazuh.company.local
       │
       ▼

DNS Configuration

View DNS Servers

cat /etc/resolv.conf

Example:

nameserver 10.10.100.11

View Resolver Status

resolvectl status

DNS Troubleshooting

nslookup

nslookup google.com

Example:

Address: 142.250.190.14

dig

Install:

sudo apt install dnsutils

Query:

dig google.com

Query Specific DNS Server

dig google.com @8.8.8.8

Hostname Resolution

Verify hostname lookup:

host google.com

Testing Port Connectivity

Sometimes hosts respond to ping but applications remain unreachable.

telnet

telnet 10.10.100.15 443

netcat (nc)

nc -zv 10.10.100.15 443

Example:

Connection succeeded

Viewing Listening Ports

ss Command

ss -tulpn

Example:

tcp LISTEN 0 128 0.0.0.0:22

Filter for Specific Port

ss -tulpn | grep 443

Legacy Method

netstat -tulpn

Network Connections

Display active sessions:

ss -ant

Example:

ESTABLISHED

Firewall Fundamentals

Linux systems commonly use the following firewall technologies:

Firewall Distribution
UFW Ubuntu
firewalld Rocky Linux
nftables Modern Linux
iptables Legacy Linux

UFW Administration

View Status

sudo ufw status

Enable Firewall

sudo ufw enable

Allow SSH

sudo ufw allow 22/tcp

Allow HTTPS

sudo ufw allow 443/tcp

Delete Rule

sudo ufw delete allow 443/tcp

firewalld Administration

View Status

sudo firewall-cmd --state

List Rules

sudo firewall-cmd --list-all

Allow Service

sudo firewall-cmd --permanent --add-service=https

Apply configuration:

sudo firewall-cmd --reload

Packet Capture Fundamentals

Packet captures assist in identifying connectivity problems.

tcpdump

Capture traffic:

sudo tcpdump -i ens18

Capture ICMP

sudo tcpdump icmp

Capture Specific Host

sudo tcpdump host 10.10.100.15

Capture Specific Port

sudo tcpdump port 443

Save Capture

sudo tcpdump -i ens18 -w capture.pcap

Open the capture later using Wireshark.

Network Performance Testing

Interface Statistics

ip -s link

Display Errors

ip -s link show ens18

Look for:

  • RX Errors
  • TX Errors
  • Dropped Packets

ARP Troubleshooting

ARP maps IP addresses to MAC addresses.

View ARP Table

ip neigh

Example output:

10.10.100.1 dev ens18 lladdr aa:bb:cc:dd:ee:ff

Flush ARP Cache

sudo ip neigh flush all

Interface Configuration Files

Ubuntu (Netplan)

Configuration location:

/etc/netplan/

Example:

network:
  version: 2
  ethernets:
    ens18:
      addresses:
        - 10.10.100.15/24
      gateway4: 10.10.100.1
      nameservers:
        addresses:
          - 10.10.100.11

Apply configuration:

sudo netplan apply

Rocky Linux

Configuration location:

/etc/NetworkManager/

Restart NetworkManager:

sudo systemctl restart NetworkManager

Common Enterprise Troubleshooting Workflow

When a server cannot reach a resource:

Step 1: Verify Interface

ip addr

Questions:

  • Is the interface UP?
  • Is an IP address assigned?

Step 2: Verify Gateway

ip route

Question:

  • Is a default route present?

Step 3: Ping Gateway

ping 10.10.100.1

Step 4: Ping External IP

ping 8.8.8.8

Success indicates routing is functioning correctly.

Step 5: Test DNS

nslookup google.com

Step 6: Verify Service Port

nc -zv server 443

Step 7: Check Firewall

sudo ufw status

sudo firewall-cmd --list-all

Step 8: Capture Traffic

sudo tcpdump

Troubleshooting Scenarios

Scenario 1: No Internet Access

ip addr
ip route
ping gateway
ping 8.8.8.8

Likely causes:

  • Interface down
  • Missing default route
  • Gateway unavailable

Scenario 2: Internet Works but Hostnames Fail

nslookup google.com
cat /etc/resolv.conf

Likely causes:

  • Incorrect DNS server
  • DNS service outage

Scenario 3: Application Unreachable

ss -tulpn
nc -zv server 443

Likely causes:

  • Service stopped
  • Firewall blocking traffic
  • Incorrect port

Scenario 4: Intermittent Connectivity

ip -s link

Look for:

  • RX errors
  • TX errors
  • Packet drops

Key Takeaways

  • The Linux networking troubleshooting process follows a layered methodology.
  • ip addr and ip route are foundational diagnostic commands.
  • Connectivity validation starts with interface status, gateway reachability, and routing verification.
  • DNS failures often appear as application connectivity problems.
  • ss, nc, and telnet help validate service availability.
  • Firewalls frequently cause connectivity issues and should always be investigated.
  • tcpdump is one of the most valuable troubleshooting tools available to Linux administrators.
  • A structured troubleshooting process reduces downtime and improves operational efficiency.
  • These networking skills are directly applicable to Linux administration, NOC operations, SOC environments, cloud platforms, and enterprise infrastructure support.
