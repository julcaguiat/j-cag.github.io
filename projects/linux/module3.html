Module 3: Linux Storage & Filesystem Management

Overview

Storage management is a critical responsibility for Linux administrators. Enterprise servers rely on properly configured disks, partitions, filesystems, logical volumes, mount points, and permissions to ensure data availability, performance, and recoverability.

This module covers Linux storage architecture, disk management, partitioning, filesystem creation, mounting, Logical Volume Manager (LVM), swap management, storage monitoring, and troubleshooting procedures commonly performed in production environments.

Learning Objectives

  • Identify storage devices and partitions.
  • Understand Linux filesystem architecture.
  • Create and manage partitions.
  • Format disks with Linux filesystems.
  • Mount and unmount filesystems.
  • Configure persistent mounts.
  • Manage Logical Volume Manager (LVM).
  • Monitor storage utilization.
  • Configure and manage swap space.
  • Troubleshoot filesystem and storage issues.
  • Perform filesystem integrity checks.

Linux Storage Architecture

Linux storage is organized into several layers: 

Physical Disk
      │
      ▼
Partition Table
      │
      ▼
Partitions
      │
      ▼
Filesystem
      │
      ▼
Mount Point
      │
      ▼
Accessible Data

Example:

Disk: /dev/sda
Partition: /dev/sda1
Filesystem: ext4
Mount Point: /data

Understanding Device Naming

Linux identifies storage devices under /dev.

Device Description
/dev/sda First SATA/SCSI disk
/dev/sdb Second SATA/SCSI disk
/dev/nvme0n1 First NVMe disk
/dev/sda1 First partition on sda
/dev/sda2 Second partition on sda
/dev/mapper Logical volume mappings

Viewing Storage Devices

List Block Devices

lsblk

Example output:

NAME    SIZE TYPE MOUNTPOINT
sda     250G disk
├─sda1    1G part /boot
├─sda2    8G part [SWAP]
└─sda3  241G part /

Display Filesystem Information

lsblk -f

Example:

NAME    FSTYPE LABEL UUID
sda1    ext4
sda2    swap
sda3    ext4

View Partition Tables

sudo fdisk -l

Understanding Filesystems

A filesystem organizes and stores data on storage devices.

Filesystem Description
ext4 Default Linux filesystem
XFS Enterprise filesystem
Btrfs Advanced copy-on-write filesystem
FAT32 Cross-platform compatibility
NTFS Windows filesystem
swap Virtual memory partition

Viewing Mounted Filesystems

Display Mounted Filesystems

mount

Display Human-Readable Usage

df -h

Example:

Filesystem      Size Used Avail Use%
/dev/sda3       240G 120G 110G 53%

Creating Partitions

Launch fdisk

Example using a secondary disk:

sudo fdisk /dev/sdb

Common fdisk Commands

Command Function
p Print partition table
n New partition
d Delete partition
w Write changes
q Quit without saving

Verify New Partition

lsblk

Example:

sdb
└─sdb1 100G part

Creating Filesystems

Format as ext4

sudo mkfs.ext4 /dev/sdb1

Format as XFS

sudo mkfs.xfs /dev/sdb1

Format as FAT32

sudo mkfs.vfat /dev/sdb1

Verify Filesystem

sudo blkid

Example:

/dev/sdb1: UUID="1234-abcd" TYPE="ext4"

Mounting Filesystems

A filesystem must be mounted before it can be used.

Create Mount Point

sudo mkdir /data

Mount Filesystem

sudo mount /dev/sdb1 /data

Verify Mount

df -h

or

mount | grep sdb1

Unmount Filesystem

sudo umount /data

or

sudo umount /dev/sdb1

Persistent Mounting with fstab

Temporary mounts disappear after reboot.

Persistent mounts are configured in: 

/etc/fstab

View Current fstab

cat /etc/fstab

Obtain UUID

sudo blkid

Example:

UUID="1234-abcd"

Add Persistent Mount

Example entry:

UUID=1234-abcd /data ext4 defaults 0 2

Validate Configuration

Before rebooting:

sudo mount -a

No output indicates success.

Understanding Logical Volume Manager (LVM)

LVM provides flexible storage management.

Benefits

  • Online volume expansion
  • Multiple disk aggregation
  • Simplified storage administration
  • Snapshot support

LVM Architecture

Physical Volume (PV)
         │
         ▼
Volume Group (VG)
         │
         ▼
Logical Volume (LV)
         │
         ▼
Filesystem

Creating Physical Volumes

Initialize a disk:

sudo pvcreate /dev/sdb1

Verify:

sudo pvs

Creating Volume Groups

Create VG:

sudo vgcreate vg_data /dev/sdb1

Verify:

sudo vgs

Creating Logical Volumes

Create a 50GB logical volume:

sudo lvcreate -L 50G -n lv_backup vg_data

Verify:

sudo lvs

Format Logical Volume

sudo mkfs.ext4 /dev/vg_data/lv_backup

Mount Logical Volume

sudo mkdir /backup
sudo mount /dev/vg_data/lv_backup /backup

Extending Logical Volumes

Increase volume by 20GB:

sudo lvextend -L +20G /dev/vg_data/lv_backup

Extend filesystem:

sudo resize2fs /dev/vg_data/lv_backup

Verify:

df -h

Swap Space Management

Swap provides virtual memory when physical RAM is exhausted.

View Swap Usage

swapon --show

or

free -h

Create Swap File

Create a 4GB swap file:

sudo fallocate -l 4G /swapfile

Secure Permissions

sudo chmod 600 /swapfile

Format Swap

sudo mkswap /swapfile

Enable Swap

sudo swapon /swapfile

Verify:

swapon --show

Persist Swap

Add to /etc/fstab:

/swapfile none swap sw 0 0

Disk Usage Monitoring

Display Filesystem Utilization

df -h

Analyze Directory Consumption

du -sh /var/log

Largest Directories

sudo du -h / | sort -rh | head -20

File Inode Management

Filesystems contain a limited number of inodes.

Check inode usage:

df -i

Example:

Filesystem Inodes IUsed IFree IUse%
/dev/sda3 1600000 300000 1300000 19%

Filesystem Integrity Checks

Filesystem corruption can occur due to:

  • Power loss
  • Hardware failure
  • Improper shutdown
  • Storage controller issues

Check Filesystem

Unmount first:

sudo umount /dev/sdb1

Run check:

sudo fsck /dev/sdb1

Force Filesystem Check

sudo fsck -f /dev/sdb1

XFS Integrity Checks

Repair XFS filesystem:

sudo xfs_repair /dev/sdb1

File Permissions Overview

Storage management includes access control.

Display permissions:

ls -l

Example:

-rw-r--r-- 1 admin admin 1024 Jun 22 report.txt

Ownership Management

Change owner:

sudo chown admin file.txt

Change owner and group:

sudo chown admin:sysadmins file.txt

Permission Management

Modify permissions:

chmod 644 file.txt
Permission Meaning
777 Full access
755 Typical directory
644 Standard file
600 Sensitive file

Storage Troubleshooting

Disk Not Detected

Verify devices:

lsblk

Check kernel detection:

dmesg | grep sd

Filesystem Not Mounting

Check UUID:

blkid

Verify fstab:

cat /etc/fstab

Test configuration:

mount -a

Disk Full

df -h

Locate large directories:

du -sh /*

Excessive Log Growth

du -sh /var/log/*

Inode Exhaustion

Verify:

df -i

Locate excessive small files:

find / -type f | wc -l

Key Takeaways

  • Linux storage management involves disks, partitions, filesystems, mount points, and permissions.
  • lsblk, fdisk, and blkid are primary disk management tools.
  • Filesystems must be formatted and mounted before use.
  • Persistent mounts are configured through /etc/fstab.
  • LVM provides flexible, enterprise-grade storage management.
  • Swap space supplements physical memory.
  • df, du, and df -i are essential monitoring tools.
  • fsck and xfs_repair help recover damaged filesystems.
  • Storage troubleshooting is a core Linux administration skill frequently encountered in enterprise operations.

Module Completion Outcome

At the completion of Module 3, you should be capable of provisioning storage devices, creating and managing filesystems, configuring persistent mounts, implementing LVM storage, managing swap space, monitoring disk utilization, and troubleshooting common Linux storage and filesystem issues encountered in enterprise server environments.
