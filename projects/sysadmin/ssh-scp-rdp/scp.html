SECURE FILE TRANSFER LAB

SCP File Transfer Guide (Windows ↔ macOS/Linux)

Comprehensive guide covering SSH enable/verification, SCP file transfers, directory movement, key-based authentication, and troubleshooting across Windows, macOS/Linux systems.

Scope

Purpose: Secure remote access and file transfer using SSH and SCP.

Environment: Windows ↔ macOS/Linux

Example IP: 192.168.1.100

Step 1 — Enable and Verify SSH

macOS / Linux

sudo systemsetup -setremotelogin on
 systemsetup -getremotelogin
 sudo lsof -iTCP -sTCP:LISTEN -n -P | grep ssh
 netstat -an | grep ".22"

Windows (PowerShell Admin)

Add-WindowsCapability -Online -Name OpenSSH.Server~~~~0.0.1.0
 Start-Service sshd
 Set-Service -Name sshd -StartupType Automatic
 Get-Service sshd

 New-NetFirewallRule -Name sshd -DisplayName "OpenSSH Server (sshd)" `
 -Enabled True -Direction Inbound -Protocol TCP -Action Allow -LocalPort 22

Step 2 — SSH Sessions (optional: try connection for full verification)

ssh user@192.168.1.100

Step 3 — SCP File Transfers

Pull File (Remote → Local)

scp user@192.168.1.100:/Users/user/Desktop/file.docx ~/Downloads/
 scp user@192.168.1.100:"C:/Users/user/Desktop/file.docx" ~/Downloads/

Push File (Local → Remote)

scp ~/Desktop/file.docx user@192.168.1.100:"C:/Users/user/Downloads/"
 scp C:\Users\user\Desktop\file.docx user@192.168.1.100:/Users/user/Downloads/

Step 4 — Directory Transfers

scp -r user@192.168.1.100:/Users/user/Desktop/Folder ~/Downloads/
 scp -r ~/Desktop/Folder user@192.168.1.100:"C:/Users/user/Downloads/Folder"
 scp -r C:\Users\user\Desktop\Folder user@192.168.1.100:/Users/user/Downloads/

Step 5 — Connectivity & Service Checks

macOS / Linux

systemsetup -getremotelogin
 sudo launchctl kickstart -k system/com.openssh.sshd
 nc -vz 192.168.1.100 22

Windows

Get-Service sshd
 Restart-Service sshd
 netstat -ano | findstr ":22"
 Test-NetConnection -ComputerName 192.168.1.100 -Port 22

Step 6 — Key-Based Authentication

Generate Key

ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519 -C "ssh-key"

Copy Key

ssh-copy-id -i ~/.ssh/id_ed25519.pub user@192.168.1.100

Manual Setup (Windows)

mkdir $env:USERPROFILE\.ssh
notepad $env:USERPROFILE\.ssh\authorized_keys
icacls $env:USERPROFILE\.ssh /grant $env:USERNAME:(R,W)
Restart-Service sshd

Step 7 — Troubleshooting

Permission denied:
 - Check username
 - Use password instead of PIN
 - Validate authorized_keys entry

 Connection issues:
 - Ensure sshd service is running
 - Confirm firewall allows port 22

 File path issues:
 - macOS: /Users/user/...
 - Windows: C:/Users/user/...

 Authentication issues:
 - Ensure PasswordAuthentication is enabled

 Network issues:
 - ping 192.168.1.100

Summary

  • SSH enabled on both endpoints
  • Port 22 accessible and open
  • SCP used for file and directory transfer
  • Key-based authentication improves security
  • Troubleshooting covers service, network, and permission issues
