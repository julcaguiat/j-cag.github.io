SSH Configuration & Key-Based Authentication (macOS ↔ Windows)

Configure SSH services and implement key-based authentication between macOS and Windows for secure passwordless remote access in a lab environment.

Scope

Purpose: Establish secure SSH connectivity between macOS and Windows systems.

Environment: macOS ↔ Windows OpenSSH Server

Example IP: 192.168.1.100

Step 1 — Enable SSH Server

macOS (Remote Login)

sudo systemsetup -setremotelogin on
 systemsetup -getremotelogin
sudo lsof -iTCP -sTCP:LISTEN -n -P | grep ssh

Windows (OpenSSH Server)

Add-WindowsCapability -Online -Name OpenSSH.Server~~~~0.0.1.0

 Start-Service sshd
 Set-Service -Name sshd -StartupType Automatic

 Get-Service sshd

Firewall Rule

New-NetFirewallRule -Name sshd -DisplayName "OpenSSH Server" `
 -Enabled True -Direction Inbound -Protocol TCP -Action Allow -LocalPort 22

Step 2 — Verify Connectivity

macOS → Windows

ping 192.168.1.100
 nc -vz 192.168.1.100 22

Windows → macOS

Test-NetConnection -ComputerName 192.168.1.100 -Port 22

Step 3 — SSH Login (Password-Based)

macOS → Windows

ssh user@192.168.1.100

Windows → macOS

ssh user@192.168.1.100

Step 4 — Generate SSH Key Pair

macOS Client

ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519 -C "mac-to-windows"

Key files:

  • ~/.ssh/id_ed25519 (private key)
  • ~/.ssh/id_ed25519.pub (public key)

Step 5 — Deploy Public Key to Windows

Option A — ssh-copy-id

ssh-copy-id -i ~/.ssh/id_ed25519.pub user@192.168.1.100

Option B — Manual Method

cat ~/.ssh/id_ed25519.pub

Windows PowerShell:

mkdir $env:USERPROFILE\.ssh
 notepad $env:USERPROFILE\.ssh\authorized_keys

Restart SSH service:

Restart-Service sshd

Step 6 — Enable Passwordless Login

ssh user@192.168.1.100

Expected result:

  • No password prompt
  • Authentication via SSH key

Step 7 — Reverse Setup (Windows → macOS)

ssh-keygen -t ed25519

macOS configuration:

mkdir -p ~/.ssh
 nano ~/.ssh/authorized_keys
 chmod 600 ~/.ssh/authorized_keys
 chmod 700 ~/.ssh
sudo systemsetup -setremotelogin on
 sudo launchctl kickstart -k system/com.openssh.sshd

Step 8 — SCP File Transfer

macOS → Windows

scp file.txt user@192.168.1.100:"C:/Users/user/Downloads/"

Windows → macOS

scp file.txt user@192.168.1.100:/Users/user/Downloads/

Troubleshooting

Permission denied → wrong user or missing key
 Connection refused → sshd not running or firewall blocked
 No route to host → network mismatch
 Key not working → permissions issue in .ssh folder
 SCP errors → incorrect Windows path format

Summary

  • SSH enabled on macOS and Windows
  • Port 22 open and reachable
  • Key-based authentication configured
  • SCP enabled for secure file transfer
