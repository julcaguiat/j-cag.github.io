VIRTUALIZATION PLATFORM DEPLOYMENT

Proxmox VE Installation on Dell Inspiron 3650

Deploy Proxmox Virtual Environment directly on Dell hardware and configure a dedicated management interface using static IP address 10.10.100.10/24.

Overview

This guide documents the deployment of Proxmox Virtual Environment (VE) on a Dell Inspiron 3650 using the official Proxmox installer.

The resulting platform provides enterprise-grade virtualization using KVM virtual machines and LXC containers suitable for networking, cybersecurity, SOC, Active Directory, and infrastructure automation labs.

The management interface will be configured with static IP address: 10.10.100.10/24

Hardware Requirements

Minimum Requirements

Component Requirement
CPU 64-bit Dual Core with VT-x / AMD-V
Memory 8 GB RAM
Storage 120 GB SSD/HDD
Network Gigabit Ethernet

Recommended Requirements

Component Recommendation
CPU Intel i5 or higher
Memory 16-32 GB RAM
Storage 500 GB SSD or larger
Network Gigabit Ethernet

Required Software

  • Proxmox VE ISO Installer
  • Rufus (Windows) or balenaEtcher (Windows/macOS/Linux)
  • 8 GB or larger USB flash drive
  • Internet connectivity

Step 1 — Download Proxmox VE

Download the latest stable ISO installer from: 

https://www.proxmox.com/en/downloads

Select:

  • Proxmox VE ISO Installer
The ISO contains the operating system, hypervisor, web interface, and required management components.

Step 2 — Create Bootable USB

Using Rufus

  1. Insert USB flash drive
  2. Launch Rufus
  3. Select Proxmox ISO
  4. Partition Scheme: GPT
  5. Target System: UEFI
  6. Click Start

Using balenaEtcher

  1. Select ISO file
  2. Select USB device
  3. Click Flash
All data on the USB device will be erased.

Step 3 — Configure BIOS

Enter BIOS Setup on the Dell Inspiron 3650 and configure:

Setting Value
Intel Virtualization Technology Enabled
VT-d Enabled
Secure Boot Disabled
Boot Mode UEFI
USB Boot Enabled

Step 4 — Install Proxmox VE

  1. Boot from USB installer
  2. Select Install Proxmox VE
  3. Accept EULA
  4. Select target disk
  5. Choose default LVM-Thin storage
  6. Configure region and keyboard layout
  7. Create root password
  8. Enter administrator email address

Step 5 — Configure Management Network

Parameter Value
Hostname SVR-01
IP Address 10.10.100.10
Subnet Mask 255.255.255.0
Gateway 10.10.100.1
DNS Server 8.8.8.8
This IP address becomes the primary management interface for the Proxmox web console.

Step 6 — Complete Installation

Click Install and allow the installer to:

  • Partition storage
  • Install Proxmox OS
  • Install KVM components
  • Configure networking
  • Deploy management services

After installation:

  1. Remove USB media
  2. Reboot system

Step 7 — Access the Web Interface

Open a browser and navigate to:

https://10.10.100.10:8006

Login Information

Field Value
User root
Realm PAM
Password Configured during installation
A certificate warning is expected because Proxmox uses a self-signed certificate by default.

Step 8 — Verification

Verify Network Connectivity

ping 10.10.100.10

Verify Cluster Status

pvecm status

Verify Interface Configuration

ip a

 ip route

Verify Core Services

systemctl status pveproxy

 systemctl status pvedaemon

Expected state: 

Active (running)

Step 9 — Update the System

apt update && apt upgrade -y

Post-Installation Tasks

  • Enable SSH access
  • Add secondary storage
  • Create Windows Server 2019 virtual machine for Active Directory
  • Create Peppermint virtual machine for ticketing services
  • Create CML virtual machine for CCNP labs
  • Configure backups

Deployment Complete

Proxmox VE is now installed and operational on the Dell Inspiron 3650.

The platform is ready to host virtual machines, containers, networking labs, Active Directory environments, SIEM platforms, and cybersecurity projects.
