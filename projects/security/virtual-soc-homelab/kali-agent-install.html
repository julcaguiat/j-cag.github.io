How to Install Wazuh Agent on Kali Linux VM

Step-by-step guide to installing and registering the Wazuh Agent on a Kali Linux VM for SIEM monitoring and endpoint visibility in a SOC home lab.

Overview

The Wazuh Agent is responsible for collecting logs, monitoring system activity, and sending telemetry data back to the Wazuh SIEM server. In this setup, Kali Linux is configured as a monitored endpoint to simulate attacker activity visibility in a SOC environment.

Requirements

  • Kali Linux VM running on VirtualBox
  • Wazuh Server (Ubuntu) already installed
  • Wazuh Dashboard access
  • Network connectivity between VM and server

Step 1 — Identify Wazuh Server IP Address

On the Ubuntu Wazuh Server, run the following command: 

ip a

Locate the inet value (example: 10.10.100.15). This is required for agent registration.

Step 2 — Deploy Wazuh Agent from Dashboard

  1. Log in to Wazuh Dashboard
  2. Navigate to Active Agents
  3. Click Deploy New Agent
  4. Select DEB amd64 package
  5. Enter Wazuh Server IP address
  6. Set Agent Name (Kali-Linux-VM)
  7. Copy installation commands

Step 3 — Install Agent on Kali Linux

  1. Open Kali Linux terminal
  2. Switch to root user
sudo su
  1. Paste and execute installation commands from dashboard

After installation, enable and start the Wazuh agent: 

sudo systemctl daemon-reload
sudo systemctl enable wazuh-agent
sudo systemctl start wazuh-agent

Verify service status: 

sudo systemctl status wazuh-agent

Status should show: active (running)

Step 4 — Verify Agent in Wazuh Dashboard

Once connected, the Kali Linux VM will appear as an active agent in the Wazuh dashboard.

Step 5 — Monitor Events and Logs

  • Security alerts
  • Process monitoring
  • /var/log system logs
  • File integrity monitoring
  • Rootkit detection events

Completion

The Kali Linux VM is now successfully enrolled as a Wazuh agent. This enables real-time monitoring of attacker simulation activity inside the SOC lab.
