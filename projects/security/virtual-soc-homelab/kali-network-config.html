Kali Linux VM — Adapter Settings & LAN IP Configuration

Configure Kali Linux with a static IP on the internal LabNet network for stable communication with pfSense and other lab VMs.

Introduction

In this part of the lab, Kali Linux is configured with a static IP address on the Internal Network (LabNet). This ensures stable communication with pfSense and other virtual machines. A fixed IP is required for reliable scanning, attack simulation, and SOC monitoring workflows.

Tip: Keep ChatGPT Handy

If NetworkManager overwrites your configuration, you can troubleshoot with:

“Kali keeps overwriting /etc/network/interfaces. How do I disable NetworkManager and make 192.168.10.2 static?”

Step 1 — Kali VM Network Adapter Configuration

  1. Power off Kali Linux VM
  2. Go to VirtualBox → Settings → Network

Adapter 1 (LAN - Internal Network)

  • Enable Network Adapter
  • Attached to: Internal Network
  • Name: LabNet
  • Promiscuous Mode: Allow All

Step 2 — Configure Static IP Address

  1. Boot Kali Linux VM
  2. Open terminal
  3. Edit network interfaces file

Run command:

sudo nano /etc/network/interfaces

Enter password: kali (or your updated password)

Static IP Configuration

auto eth0
iface eth0 inet static
    address 192.168.10.2
    netmask 255.255.255.0
    gateway 192.168.10.1
    dns-nameservers 192.168.10.1 1.1.1.1

Save file: CTRL + X → Y → Enter

Step 3 — Restart Networking

sudo systemctl restart networking
ip a

Kali Linux is now assigned static IP: 192.168.10.2

Step 4 — Connectivity Test

Run the following commands:

ping -c 4 192.168.10.1
ping -c 4 google.com

First verifies pfSense connectivity. Second verifies internet access.

Step 5 — Access pfSense GUI

  • Open browser in Kali
  • Navigate to: https://192.168.10.1:8443

Completion

Kali Linux is now configured with a static IP (192.168.10.2) and integrated into the LabNet internal network. It is fully reachable by pfSense and ready for security testing workflows.
