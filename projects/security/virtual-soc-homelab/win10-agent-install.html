How to Install Wazuh Agent on Windows 10 VM

Step-by-step guide to installing and registering the Wazuh Agent on a Windows 10 virtual machine for SIEM monitoring in a SOC home lab environment.

Overview

The Wazuh Agent collects security telemetry such as event logs, system activity, and file integrity changes from endpoints and forwards them to the Wazuh Manager for centralized analysis.

In this guide, a Windows 10 VM is configured as a monitored endpoint inside the SOC lab environment.

Requirements

  • Windows 10 VM running on VirtualBox
  • Wazuh Server (Ubuntu) already deployed
  • Wazuh Dashboard access
  • Network connectivity between systems

Step 1 — Identify Wazuh Server IP Address

On the Ubuntu Wazuh Server, run:

ip a

Locate the inet address (example: 10.10.100.15). This will be used during agent setup.

Step 2 — Deploy Windows Agent from Wazuh Dashboard

  1. Log in to Wazuh Dashboard
  2. Navigate to Active Agents
  3. Click Deploy New Agent
  4. Select MSI (32/64-bit)
  5. Enter Wazuh Server IP address
  6. Set Agent Name (Windows10-VM)
  7. Copy generated installation commands

Step 3 — Install Agent on Windows 10

  1. Open Windows 10 VM
  2. Launch PowerShell as Administrator
  3. Paste installation commands from Wazuh Dashboard

After installation, start the Wazuh service:

NET START WazuhSvc

Step 4 — Verify Agent in Wazuh Dashboard

Once successfully installed, the Windows 10 VM will appear as an active agent in the Wazuh dashboard.

Step 5 — Monitor Logs and Events

  • Windows Event Logs (logins, failures, policy changes)
  • File Integrity Monitoring (FIM)
  • Registry modifications
  • Service and process monitoring

Completion

The Windows 10 VM is now successfully integrated with Wazuh SIEM and actively monitored. This completes endpoint visibility across both Linux and Windows systems in the SOC lab.
