🖧 Why I Use NAT + Internal Network (and Not Bridged or NAT Network) in My pfSense Home Lab

Understanding VirtualBox networking modes and why pfSense must sit in the middle of all traffic in a SOC lab.

Overview

VirtualBox provides multiple networking modes such as NAT, NAT Network, and Bridged Adapter. While all provide internet access, they behave very differently in a security lab environment.

If your goal is to practice firewalling, routing, and traffic inspection using pfSense, network design becomes critical.

⚡ NAT Network

NAT Network creates a private subnet managed by VirtualBox where VMs can communicate and access the internet. 

[ Kali ] ─┐
          │
[ Win10 ] ─┘──> VirtualBox NAT Network ──> Host ──> Internet
  • ✔ VMs can communicate with each other
  • ✔ Internet access available
  • ❌ pfSense is bypassed if not explicitly in path

This is useful for quick testing but not suitable for firewall simulation.

🌍 Bridged Adapter

Bridged mode connects VMs directly to your physical network, making them appear as real devices on your LAN. 

[ Kali ] ──> Home Router ──> Internet
[ Win10 ] ──> Home Router ──> Internet
  • ✔ VM behaves like a physical machine
  • ✔ Can access LAN devices directly
  • ❌ Bypasses pfSense unless carefully integrated
  • ❌ Reduces isolation for security testing

🔒 Why I Avoid NAT Network and Bridged in My Lab

Both NAT Network and Bridged modes allow VMs to reach the internet without passing through pfSense. This breaks the purpose of building a firewall-based SOC lab.

Preferred lab architecture:

[ Kali ] ─┐
[ Win10 ] ─┘──> pfSense LAN (Internal Network)
                     │
                     ▼
              pfSense WAN (NAT)
                     │
                     ▼
                  Internet
  • ✔ pfSense becomes the central gateway
  • ✔ All traffic is inspectable and controllable
  • ✔ Enables firewall rule testing
  • ✔ Supports realistic SOC simulation

✅ Key Takeaway

  • NAT Network: Easy internet + VM communication, but bypasses pfSense
  • Bridged: Joins real network, but removes isolation and control
  • NAT + Internal Network: Proper SOC design with pfSense as firewall gateway

Summary

A proper SOC home lab depends on traffic flow control. By forcing all VM traffic through pfSense, you gain visibility, control, and realistic security simulation capability.
