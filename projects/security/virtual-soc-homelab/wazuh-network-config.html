Ubuntu-Wazuh VM — Adapter Settings & LAN IP Configuration

Configure Ubuntu Wazuh as a central SIEM server with NAT + Internal Network connectivity for full lab monitoring.

Introduction

In this part of the lab, the Ubuntu Wazuh VM is configured with two network adapters: NAT for external connectivity and Internal Network (LabNet) for communication with pfSense, Kali Linux, and Windows endpoints.

This setup enables Wazuh to function as a centralized SIEM platform for log collection, alerting, and security monitoring across the entire home lab environment.

Step 1 — Ubuntu Wazuh Network Adapter Configuration

  1. Power off Ubuntu Wazuh VM
  2. Go to VirtualBox → Settings → Network

Adapter 1 (WAN - NAT)

  • Enable Network Adapter
  • Attached to: NAT
  • Promiscuous Mode: Allow All
  • Configure Port Forwarding

Port Forwarding Rules

  • Wazuh-API → TCP → 5500 → 192.168.10.4:5500
  • Wazuh-Dashboard → TCP → 443 → 192.168.10.4:443

Adapter 2 (LAN - Internal Network)

  • Enable Network Adapter
  • Attached to: Internal Network
  • Name: LabNet
  • Promiscuous Mode: Allow All

Step 2 — Configure Static IP (Netplan)

Commands:

sudo su
cd /
cd /etc/netplan
ls
nano 50-cloud-init.yaml

Netplan Configuration

network:
  version: 2
  ethernets:
    enp0s3:
      dhcp4: true
    enp0s8:
      dhcp4: false
      addresses:
        - 192.168.10.4/24
      routes:
        - to: 0.0.0.0/0
          via: 192.168.10.1
      nameservers:
        addresses:
          - 1.1.1.1
          - 8.8.8.8

Save file: CTRL + X → Y → Enter

Step 3 — Apply Configuration & Test Connectivity

sudo netplan apply
ping -c 4 192.168.10.1
ping -c 4 google.com
ip a

Ubuntu Wazuh is now assigned static IP: 192.168.10.4

Step 4 — Access Wazuh Dashboard

  • Open browser from Kali or Windows VM
  • Navigate to: https://192.168.10.4:443

Completion

Ubuntu Wazuh is now fully configured as a SIEM server with static IP 192.168.10.4. It is integrated into the LabNet network and ready to collect logs from all endpoints.
