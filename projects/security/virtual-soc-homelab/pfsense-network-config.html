pfSense VM — Adapter Settings & LAN IP Configuration

Configure NAT and Internal Network adapters for pfSense and set a static LAN IP for lab segmentation.

Introduction

In this part of the lab, we configure pfSense with two network adapters: NAT for external connectivity and Internal Network for LAN communication. pfSense will act as the central firewall and gateway between Internet and internal systems. DHCP is disabled to maintain static IP consistency across the lab.

Tip: Keep ChatGPT Handy

If pfSense fails to boot or LAN is unreachable, you can troubleshoot using prompts like:

"pfSense VirtualBox NAT + Internal Network not reachable at 192.168.10.1. Console output: ... What should I check?"

Step 1 — Configure pfSense Network Adapters

  1. Power off pfSense VM
  2. Go to VirtualBox → Settings → Network

Adapter 1 (WAN - NAT)

  • Enable Network Adapter
  • Attached to: NAT
  • Promiscuous Mode: Allow All
  • Open Port Forwarding

Add Port Forwarding Rule:

  • Name: pfSense
  • Protocol: TCP
  • Host Port: 8443
  • Guest IP: 10.0.2.15
  • Guest Port: 8443

Adapter 2 (LAN - Internal Network)

  • Enable Network Adapter
  • Attached to: Internal Network
  • Name: LabNet
  • Promiscuous Mode: Allow All

Step 2 — Configure LAN Static IP

  1. Boot pfSense VM
  2. Select Option 2: Set interface IP

Configuration Inputs:

  • Interface: LAN (em1)
  • IPv4: Static
  • IP Address: 192.168.10.1
  • Subnet: /24
  • DHCP: Disabled
  • IPv6: Disabled

Access pfSense Web UI

Once configured, access the firewall using:

https://192.168.10.1:8443

  • Username: admin
  • Password: admin

If access fails, disable packet filtering via shell: pfctl -d

Completion

pfSense is now configured with WAN and LAN interfaces. The LAN subnet 192.168.10.0/24 will serve as the core internal lab network.
