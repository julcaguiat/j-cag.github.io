Windows 10 VM — Adapter Settings & LAN IP Configuration

Configure Windows 10 as an internal SOC endpoint with static IP addressing for reliable communication inside LabNet.

Introduction

In this part of the lab, the Windows 10 VM is configured with a static IP address on the Internal Network (LabNet). This ensures stable communication with pfSense, Kali Linux, and the Wazuh SIEM server.

Tip: Keep ChatGPT Handy

If Windows shows “Unidentified Network” or loses connectivity:

“Windows 10 VM on VirtualBox Internal Network (LabNet) cannot reach gateway 192.168.10.1. Here are my IPv4 settings: <details>. What should I fix?”

Step 1 — Windows 10 VM Adapter Configuration

  1. Power off Windows 10 VM
  2. Go to VirtualBox → Settings → Network

Adapter 1 (LAN - Internal Network)

  • Enable Network Adapter
  • Attached to: Internal Network
  • Name: LabNet
  • Promiscuous Mode: Allow All

Step 2 — Configure Static IP Address (GUI Method)

  1. Boot Windows 10 VM
  2. Open Settings → Network
  3. Click Change adapter options
  1. Right-click Ethernet → Properties
  2. Select Internet Protocol Version 4 (TCP/IPv4)
  3. Click Properties

Static IP Configuration

  • IP Address: 192.168.10.3
  • Subnet Mask: 255.255.255.0
  • Default Gateway: 192.168.10.1

DNS Configuration

  • Preferred DNS: 192.168.10.1
  • Alternate DNS: 1.1.1.1

Step 3 — Verify Configuration

Run the following command in Command Prompt:

ipconfig

Windows 10 VM is now assigned: 192.168.10.3

Step 4 — Connectivity Tests

Test internal and external connectivity:

ping 192.168.10.1
ping google.com

Step 5 — Access Services

  • pfSense GUI: https://192.168.10.1:8443
  • Wazuh Dashboard: https://192.168.10.4:443

Completion

The Windows 10 VM is now fully integrated into the SOC lab as an endpoint. It can communicate with pfSense, Kali Linux, and Wazuh SIEM for monitoring and testing.

You now have a fully functional SOC home lab with:

  • pfSense firewall (traffic control)
  • Wazuh SIEM (log analysis)
  • Kali Linux (attack simulation)
  • Windows 10 endpoint (target system)
