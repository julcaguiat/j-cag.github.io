Sysmon Installation & Configuration

Deploy Sysmon on a Windows 11 endpoint and integrate it with Wazuh SIEM to provide enhanced endpoint telemetry, process visibility, network monitoring, and detection engineering capabilities.

Overview

Sysmon (System Monitor) is a Microsoft Sysinternals utility that extends native Windows logging by recording detailed endpoint activity such as process creation, network connections, file modifications, image loads, and DNS requests.

When integrated with Wazuh, Sysmon provides high-fidelity telemetry used for threat hunting, incident response, detection engineering, and SOC operations.

Prerequisites

  • Windows 11 Pro VM
  • Administrator privileges
  • Internet access
  • Wazuh Agent installed (optional but recommended)
  • PowerShell access

Step 1 — Download Sysmon

Download Sysmon from Microsoft's Sysinternals Suite.

Recommended extraction directory: 

C:\Tools\Sysmon\

After extraction, verify the following files exist:

  • Sysmon.exe
  • Sysmon64.exe

Step 2 — Download a Sysmon Configuration

Sysmon requires an XML configuration file that defines which events will be collected.

A commonly used baseline is the SwiftOnSecurity Sysmon configuration.

Save the configuration as: 

C:\Tools\Sysmon\sysmonconfig.xml
The SwiftOnSecurity configuration provides a strong balance between visibility and event noise reduction and is widely used throughout the security community.

Step 3 — Install Sysmon

Open PowerShell as Administrator and navigate to the Sysmon directory. 

cd C:\Tools\Sysmon

.\Sysmon64.exe -accepteula -i sysmonconfig.xml

This command:

  • Accepts the Sysinternals EULA
  • Installs the Sysmon service
  • Installs the Sysmon driver
  • Applies the XML configuration
  • Begins event collection immediately

Step 4 — Verify Sysmon Service

Verify the service is running. 

Get-Service Sysmon64

Expected output: 

Status   Name      DisplayName
Running  Sysmon64  System Monitor service

Step 5 — Verify Event Generation

Open Event Viewer and navigate to: 

Event Viewer
→ Applications and Services Logs
→ Microsoft
→ Windows
→ Sysmon
→ Operational

Generate activity such as:

  • Opening PowerShell
  • Launching applications
  • Browsing websites
  • Running command-line tools

New Sysmon events should appear immediately.

Step 6 — Configure Wazuh Agent Collection

Verify the Wazuh Agent is configured to monitor the Sysmon Event Channel.

Edit: 

C:\Program Files (x86)\ossec-agent\ossec.conf

Add or verify the following configuration: 

<localfile>
  <location>Microsoft-Windows-Sysmon/Operational</location>
  <log_format>eventchannel</log_format>
</localfile>

Restart the Wazuh agent: 

Restart-Service WazuhSvc

Step 7 — Validate Log Forwarding

On the Wazuh Manager, search for incoming Sysmon events. 

grep Sysmon /var/ossec/logs/archives/archives.json

Typical fields include:

  • providerName: Microsoft-Windows-Sysmon
  • eventID: 1
  • eventID: 3
  • eventID: 7
  • eventID: 22

Step 8 — Common Sysmon Event IDs

  • Event ID 1 — Process Creation
  • Event ID 2 — File Creation Time Changed
  • Event ID 3 — Network Connection
  • Event ID 7 — Image Loaded
  • Event ID 10 — Process Access
  • Event ID 11 — File Creation
  • Event ID 22 — DNS Query

Step 9 — Troubleshooting

Sysmon Not Running

Get-Service Sysmon64

Verify Current Configuration

Sysmon64.exe -c

Wazuh Not Receiving Logs

netstat -ano | findstr 1514
  • Verify Wazuh agent is running
  • Verify manager connectivity
  • Verify EventChannel configuration exists

Decoder Verification

windows_eventchannel
/var/ossec/ruleset/rules/0595-win-sysmon_rules.xml

Configuration Complete

Sysmon is now installed and actively collecting endpoint telemetry from the Windows 11 endpoint.

With Wazuh integration enabled, process activity, network connections, DNS requests, and file events are now available for centralized monitoring, alerting, threat hunting, and detection engineering workflows.

Proper Sysmon tuning is important. Excessive event collection can generate significant storage consumption and alert fatigue, while insufficient logging may reduce visibility during investigations.
