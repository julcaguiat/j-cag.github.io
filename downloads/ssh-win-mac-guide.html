🔑 SSH Connection Cheat Sheet: Windows ↔ Mac/Linux

Quick reference for setting up SSH connections between Windows, macOS, and Linux systems.

Overview

SSH (Secure Shell) allows secure remote access between systems. The command syntax is consistent across platforms: 

ssh username@IPaddress

The main difference between systems is enabling the SSH server on the target machine.

Windows (SSH Server Setup)

  1. Open Settings → Apps → Optional Features
  2. Click Add a Feature
  3. Install OpenSSH Server

Start SSH service (PowerShell as Admin):

Get-Service sshd
Start-Service sshd

Ensure Windows Firewall allows SSH connections when prompted.

Linux (SSH Server Setup)

Install OpenSSH Server:

sudo apt install openssh-server

Enable SSH service:

sudo systemctl enable --now ssh

macOS (SSH Server Setup)

  • Go to System Preferences
  • Select Sharing
  • Enable Remote Login

SSH Command Examples

From Windows → Linux/macOS:

ssh bob@192.168.1.50

From Linux/macOS → Windows/Linux:

ssh alice@192.168.1.50
  • First connection may prompt for host key confirmation → type yes
  • Then enter the password for the target user

Notes & Tips

  • Use correct username on the target system
  • Find IP using ipconfig (Windows) or ip a (Linux)
  • Use exit to close SSH session
  • First-time connections will always verify host key

Summary

SSH provides secure remote access between systems and is a foundational tool in networking, system administration, and cybersecurity workflows.
