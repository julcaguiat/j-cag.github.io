SSH and SCP Guide: Windows ↔ macOS/Linux

Secure remote access and file transfer using SSH and SCP across Windows, macOS, and Linux systems.

Overview

SCP (Secure Copy Protocol) uses SSH to securely transfer files between systems. This guide assumes both machines are on the same local network.

Note: If SSH is already enabled from the previous guide, skip the setup section.

1. Enable and Verify SSH

macOS / Linux

sudo systemsetup -setremotelogin on
systemsetup -getremotelogin

Check SSH listening ports:

sudo lsof -iTCP -sTCP:LISTEN -n -P | grep ssh
netstat -an | grep ".22"

Windows (PowerShell as Admin)

Add-WindowsCapability -Online -Name OpenSSH.Server~~~~0.0.1.0
Start-Service sshd
Set-Service -Name sshd -StartupType Automatic
Get-Service sshd

Firewall rule:

New-NetFirewallRule -Name sshd -DisplayName "OpenSSH Server (sshd)" `
-Enabled True -Direction Inbound -Protocol TCP -Action Allow -LocalPort 22

2. SCP File Transfers

Mac/Linux → Windows

scp user@192.168.1.10:/Users/user/Desktop/file.txt /Users/macuser/Downloads/

Windows → Mac/Linux

scp C:\Users\user\Desktop\file.txt user@192.168.1.10:/Users/macuser/Downloads/

Windows Pull Example

scp macuser@192.168.1.10:/Users/macuser/Desktop/file.txt C:\Users\windowsuser\Downloads\

3. Recursive Folder Transfers

scp -r user@192.168.1.10:/Users/user/Desktop/Folder /Users/macuser/Downloads/

scp -r C:\Users\user\Desktop\Folder user@192.168.1.10:/Users/macuser/Downloads/Folder

4. Quick Troubleshooting

macOS / Linux

systemsetup -getremotelogin
sudo launchctl kickstart -k system/com.openssh.sshd
nc -vz 192.168.1.10 22

Windows

Get-Service sshd
Restart-Service sshd
netstat -ano | findstr ":22"
Test-NetConnection -ComputerName 192.168.1.10 -Port 22

5. Key-Based Authentication

Generate Key (Mac/Linux)

ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519 -C "mac-to-win-key"

Copy Key to Windows

ssh-copy-id -i ~/.ssh/id_ed25519.pub user@192.168.1.10

Manual Method

cat ~/.ssh/id_ed25519.pub

Paste into:

C:\Users\user\.ssh\authorized_keys

Restart SSH service:

Restart-Service sshd

6. Common Issues

  • Permission denied: verify username and password
  • Connection refused: SSH service or firewall issue
  • No route to host: network mismatch
  • scp path errors: use correct Windows path format
  • Key issues: verify authorized_keys permissions
  • Interrupted transfers: use rsync for resume support

Summary

SSH and SCP form the foundation of secure remote administration and file transfer. These tools are essential in networking, DevOps, and cybersecurity workflows.
